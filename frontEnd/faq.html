Questions You May Have

How does it work?

The capturing device (hopefully a Raspberry Pi) will run on a specific wireless network on campus. It will capture ‘Every’ packet on that interface from all the users connected to it, will scrape basic information from every packet(destination, time, size of packet, etc), analyze them, and store them on a JSON file.

The JSON file will be sent to the server hosting the website, where the data will be displayed in multiple ways. The update time interval is 10 seconds at the moment.

Is my privacy being invaded?

No. Although we might see packets coming from your device, these packets are encrypted. We see the header of the packet, which contains metadata such as source, destination, browser version, timestamps, etc. We also don’t keep track of IP addresses.

Is this capturing ALL the traffic at hunter college?

No. The script will only capture the traffic flowing through the repeater where our capturing device is connected. If multiple repeaters belong to the same subnet (which is the case on many places in the library), then our capturing range increases to multiple repeaters. But it is mostly based on physical location.

Also, the fact that you might be next to the capturing device does not guarantee that you are connected to the same repeater. If you came from direction A, and I came from direction B, and we meet at C, I might connect to C, but your device might remain connected to A, or vice versa.

Will this capture all the websites i, or any other user goes to ?

Yes, and No. Some websites are hosted in an environment that might return a different hostname than what the GET request is when you attempt to access it.

r For instance, if you are hosting a website on AWS (Amazon Web Services), and your website is called ‘xyz.com’, our capturing script might see this request as an AWS request, making xyz.com pretty much invisible in our data. Our other graphing tool should be able to capture every single website you type in your browser though, as it is capturing DNS queries (GET requests).

Can this script be used at another location?

No. If the network is open, open in the sense that it only requires you to agree to some terms (Starbucks), or enter a username and password on a web portal AFTER BEING ALREADY CONNECTED TO THE NETWORK (Hunter Campus), then it will work.

If the network is password protected, whether WEP, WPA, WPA2, 802.1x, etc, then it will not work. There are ways to set it up so that you can decrypt packets on these specific protocols as they come in, but it is not needed at Hunter College.
